## 2. Что удалить, отключить или спрятать (под «сухую» CRM без следов ИИ)

### 2.1 Жёстко связано с ИИ / агентами

- Зависимости в `package.json`: `openai`, `@openai/agents`, `ai`, `@modelcontextprotocol/sdk`, `mcp-handler`, `@mendable/firecrawl-js`, `e2b` (песочница агента в README).
- **Маршруты:** `app/api/mcp/[transport]/route.ts` — MCP-сервер для внешних агентов.
- **Админка:** `app/[locale]/(routes)/admin/llm-keys/page.tsx`.
- **Профиль:** вкладка LLM (`/profile?tab=llms` — уточнить в коде).
- **Enrichment UI:** `crm/contacts/enrichment`, `campaigns/targets/enrichment`, API `.../enrich`, `.../enrich-bulk`.
- **Inngest-функции:** `enrich-*.ts`, `embed-*.ts`, `embed-backfill.ts`, `emails/embed-email.ts`, `documents/enrich-document.ts` — векторизация и обогащение.
- **Prisma:** модели `crm_Embeddings_*`, `EmailEmbedding`, `crm_Contact_Enrichment`, `crm_Target_Enrichment` — при полном отказе от поиска по смыслу можно оставить таблицы пустыми или миграцией выпилить (риск: много правок по FK).
- **`public/SKILL.md`** — документация для MCP; для GERS не нужна в проде.
- **Переменные окружения:** `OPENAI_API_KEY`, `FIRECRAWL_API_KEY` — не задавать; убрать из обязательной документации для внутренней сборки.

Практичный путь: **не удалять БД-модели в первом коммите**, а отключить роуты/UI/фоновые джобы и ключи — меньше поломать миграции.

### 2.2 Кампании и маркетинг (если не нужны агентству в v1)

- Разделы `/campaigns/*`, Resend webhooks, Inngest `campaigns/*`.
- Оставить, если планируете **email-рассылки по лидам**; иначе скрыть из навигации и не настраивать `RESEND_*`.

### 2.3 Прочее «тяжёлое»

- **Fulltext / semantic search** — `fulltext-search` роуты + embeddings; для узкой CRM можно скрыть и не запускать embed jobs.
- **Projects** (`/projects/*`) — отдельный модуль PM; для чистой продажной CRM убрать из меню или не использовать.
- **Databox** — если не подключаете внешнюю аналитику.
- **Rossum** (инвойсы) — env в `.env.example`; не использовать, не задавать ключи.
- **MongoDB** в зависимостях — для миграции со старой версии; на зелёном поле не нужен.

## 3. Что доработать под GERS

1. **Воронка и поля сделки** — завести стадии под ваш процесс (входящий → КП → договор и т.д.); добавить кастомные поля или использовать `description` / продукты: тип проекта (сайт / веб / мобайл / комплекс), диапазон бюджета как у формы на сайте.
2. **Сайт → CRM** — расширить `create-lead-from-web` или добавить `create-opportunity-from-web`: сразу создавать **Opportunity** + Contact + привязка к Account, дедуп по email/Telegram.
3. **Калькулятор gers.agency** — отдельный webhook с суммой и выбранными опциями в JSON.
4. **Локализация** — `locales/ru.json`, дефолтная локаль `ru`, правка копирайта («NextCRM» → бренд внутренний).
5. **Навигация** — один конфиг меню: только CRM + отчёты + задачи (+ почта по желанию).
6. **Telegram** — серверный endpoint: приём апдейтов бота, запись в `crm_Activities` или отдельная таблица сообщений + связь с Opportunity.
7. **Телефония** — webhook от провайдера → запись активности «звонок» + ссылка на запись.
8. **Дизайн** — тема shadcn, логотип, убрать маркетинговые блоки README из приложения (если где-то встроены).

## 4. Технические риски

| Риск | Комментарий |
|------|-------------|
| Node 22+ | На старых серверах поднять версию или использовать Docker из репо |
| Inngest | Нужен аккаунт/ключ для фоновых задач (почта, старые embed); при «обрезанной» CRM — отключить неиспользуемые функции, оставить минимум |
| PostgreSQL + pgvector | Расширение для embeddings; если embeddings не используете — проверить, что миграции не падают без расширения |
| Объём кодовой базы | Полное удаление ИИ-слоя — десятки файлов; оценка: несколько дней аккуратного рефакторинга + регрессия |

## 5. Итоговая рекомендация

**Брать NextCRM как основу — разумно:** MIT, стек как у вас (Next, Prisma, PostgreSQL), уже есть сделки по стадиям, контакты, аккаунты, активности, отчёты, заявка с сайта по токену, почтовый клиент.

**Первый спринт:** поднять Docker-стенд, завести одну воронку GERS, подключить форму к `create-lead-from-web` (или форку endpoint), **скрыть** admin LLM, MCP, enrichment, fulltext, campaigns/projects из UI, не задавать AI-ключи.

**Второй спринт:** Telegram + доработка полей; затем вырезание мёртвого кода и зависимостей (отдельный PR).
